Expected Behavior

  • Both embed methods should display the PDF inline, without needing to open any new tabs or adding an entry to the "Downloads" window.
      • The direct link should open the PDF in a new tab, without needing to add an entry to the "Downloads" window.
  • The download link should enact the user's preferred download behavior for the PDF file type. Either opening a download dialogue, automatically saving the file to the download directory, or automatically opening the file in the user's software of choice.

Actual Behavior

In most browsers (Chrome, Edge, etc.), all four methods behave as expected.

If we were to replace the PDF with some other file that Firefox can display, like a PNG, all four methods would behave as expected even in Firefox.

But exclusively for PDFs in current versions of Firefox, it is impossible to make this webpage behave as expected.

If PDF behavior is set to "Open in Firefox", the embed, iframe, and direct link behave correctly, but clicking the download link will download the file to the temp folder (without opening a download dialogue) and open it in a new tab. It is impossible for the webpage to open a download dialogue with this setting enabled.
